Webアプリ仕様書：mAP Visualizer (仮称)
1. プロジェクト概要
アプリ名: mAP Visualizer（任意で変更してください）

目的: 物体検出モデルの評価指標であるIoU、PR曲線（適合率-再現率曲線）、AP、mAPの計算の仕組みを、ブラウザ上の直感的な操作（バウンディングボックスの配置と信頼度の調整）を通じて学習・理解できるツールを提供する。

ターゲット: 機械学習の初学者、学生、AIエンジニア、AIモデルの評価方法を直感的に理解したい人。

マネタイズ: 完全無料で一般公開し、Google AdSenseによる広告収入を目指す。

2. システム構成・技術スタック
バックエンド（サーバー）を持たない、完全なフロントエンド・シングルページアプリケーション（SPA）として構築する。

ホスティング: Cloudflare Pages

フレームワーク: React + TypeScript (Vite環境)

描画（キャンバス）: React Konva (react-konva) - 画像上のボックス描画・ドラッグ操作に使用

グラフ描画: Recharts または Chart.js - PR曲線のリアルタイム描画に使用

スタイリング: Tailwind CSS

3. 画面構成（UIレイアウト）
画面は主に4つのエリアで構成する。

ヘッダー: アプリ名、言語切り替え（将来用）、使い方の簡単な説明。

メインワークスペース（左側/中央）:

キャンバスエリア: 背景画像と、その上にオーバーレイされるGT（Ground Truth）およびPredictのバウンディングボックスを表示・編集するエリア。

画像選択コントロール: 「自分の画像をアップロード」または「COCOデータセットのサンプル（プリセット）から選択」を切り替えるボタン。

サイドバー（右側）:

ボックス一覧＆信頼度スライダー: 作成したPredictボックスの一覧と、それぞれの信頼度（Confidence）を0.0〜1.0で調整できるスライダー。

パラメータ設定: IoUの閾値（例: 0.5）を設定するスライダー。

指標表示パネル: リアルタイム計算された Precision, Recall, AP, mAP のスコアを大きく表示。

分析・解説エリア（下部）:

PR曲線グラフ: Predictボックスを追加したり信頼度を動かすと、リアルタイムで変化する折れ線グラフ。

解説テキスト（SEO＆広告対策）: mAPの計算手順、TP/FP/FNの定義、計算式などを図解入りで解説するセクション。（※ここにAdSense広告の枠を配置する）

4. 機能要件
4.1. 背景画像とデータセット機能
カスタム画像アップロード: ユーザーがローカルの画像（JPEG/PNG）を読み込み、キャンバスの背景として設定できる。

COCOサンプルプリセット: アプリ内にあらかじめ用意されたCOCOデータセットの画像（3〜5枚程度）を選択できる。

GTの自動読み込み: サンプル画像が選択された場合、あらかじめアプリ内に持たせておいたアノテーションデータ（JSON等）を読み込み、GTボックスを自動でキャンバス上に配置する。

4.2. バウンディングボックスの操作機能
モード切替: 「GT追加モード」と「Predict追加モード」を切り替えられる。

描画: キャンバス上をドラッグして、新しいボックスを作成できる。（GTは緑色、Predictは赤色など、視覚的に区別する）

編集・削除: 配置したボックスは、後から移動（ドラッグ）、サイズ変更（リサイズ）、削除ができる。

クラス分類（拡張要件）: 最初のバージョンは「1クラス（例: catのみ）」でAPを計算する仕様で良いが、将来的に複数クラス（cat, dog）を持たせて、各クラスのAPの平均である「mAP」を計算できるようにする。

4.3. 指標のリアルタイム計算ロジック
キャンバス上のボックスや、サイドバーの信頼度が変更されるたびに、以下の計算を即座に（数ミリ秒で）実行し、画面に反映する。

IoU（Intersection over Union）の計算: 全てのPredictボックスとGTボックスの組み合わせについて、重なり面積の割合を計算する。

マッチング (TP / FP / FN の判定):

設定されたIoU閾値（例: 0.5）以上の重なりを持つPredictを True Positive (TP) とする。

閾値未満、またはGTが存在しない場所のPredictを False Positive (FP) とする。

どのPredictともマッチしなかったGTを False Negative (FN) とする。

ソート: Predictボックスを、ユーザーが設定した「信頼度（Confidenceスコア）」の降順に並び替える。

Precision / Recall の計算: 降順に並べたリストを上から順に評価し、累積の適合率（Precision）と再現率（Recall）を計算する。

APの算出: 計算したリストを元にPR曲線を描き、その下の面積（例: 11点補間法や全点補間法）を計算してAPとする。

5. 静的アセット（データ）の持ち方
バックエンドサーバーがないため、COCOのサンプル画像とGTデータはアプリ内に同梱する。

/public/samples/ ディレクトリ内に、画像（image1.jpg）と、それに対応するGT座標データを持ったJSON（image1_gt.json）を数セット配置しておく。

JSONの構造例:

[
{"id": 1, "class": "dog", "x": 100, "y": 150, "width": 200, "height": 300},
{"id": 2, "class": "cat", "x": 400, "y": 200, "width": 150, "height": 150}
]


## 6. 今後の展開とマネタイズに向けた注意事項
* **SEO対策とコンテンツの充実:** ツール単体ではGoogle AdSenseの審査（有用性の低いコンテンツ）で弾かれる可能性が高いため、トップページ下部や別ページに「物体検出の評価指標に関する質の高い解説記事」を必ず記述する。
* **スマホ対応（レスポンシブ）:** Canvasを使ったツールはスマホで操作しづらいため、基本はPC向けとするが、スマホでアクセスしたユーザーには「PCでの閲覧を推奨します」というメッセージを出すか、簡易的な閲覧モードを提供する。
* **プライバシーポリシー・免責事項の設置:** フッターに必ず設置する（AdSense審査に必須）。
